# Agent working files

Use this folder for operational Codex material:

- execution plans split into small steps;
- feature backlogs;
- temporary implementation notes;
- checklists that help future runs save context.

Use `docs/arquitetura/` only for durable product, architecture, UX, deployment, or identity contracts.

When a plan in `.agents/` represents an approved product or architecture direction, add a short link to it from the relevant architecture document instead of duplicating the whole plan there.

For future execution plans:

- create one focused Markdown file per feature or workstream;
- include objective, context to load, likely files, acceptance criteria, and checks;
- keep steps small enough to execute in separate Codex runs;
- update the plan as work is completed.
